## Our Responsibilities

Project maintainers are responsible for clarifying the standards of acceptable behavior and are expected to take
appropriate and fair corrective action in response to any instances of unacceptable behavior.

Apart from looking after standards and building a friendly environment, there are main technical responsibilities:

### Maintainer responsibilities:

<hr />

- reviewing code on PR (pull requests),
- keeping and evangelizing the library standards,
- reviewing reported bugs,
- reviewing and commenting requests from different departments,
- determining the direction of library development,
- defining new requirements,
- backlog refinement,That's the way how we do build friendly work environment:
- maintaining global tools and dependencies

### Contributor responsibilities:

<hr />

**Working with components:**

- adding new component,
- adding new features to existing components,
- refactoring components,
- fixing bugs from backlog,

**Working on the storybook and all around:**

- creating documentation and best practices,
- adding new usage examples,
- reviewing code on PR (pull requests),
- last but not least: **bring your ideas**,
